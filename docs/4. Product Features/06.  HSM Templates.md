**HSM (Highly Structured Message) templates are pre-approved message formats required for sending outbound messages to contacts outside the 24-hour WhatsApp session window.**

Templates help ensure compliance with WhatsApp's policies and are commonly used to send notifications, alerts, or support messages. They can include placeholders for personalised content and are essential for initiating conversations once the 24-hour session window has expired.

**Session Window** — A session window is the 24-hour period following a user's initial interaction with your business on WhatsApp. During this time you can send unlimited messages without additional costs. If the user doesn't respond within that window, you must wait for them to initiate contact again, or use a pre-approved template to reach out.

## When to Use HSM Templates

## When to Use HSM Templates

**Use them for:**

- For initiating conversation with the contacts after the 24-hour session window is closed, such as sending reminders or important follow-ups.

- For broadcasting standardized updates, like activity schedules, weather alerts, crop advisories, appointment confirmations, service announcements,programs and events

- Please Note: The next node after an HSM must always be Wait for Response.This ensures the system waits for user interaction before continuing the flow.

Here's a simple explanation with examples:

**Static URL**

- A Static URL is a fixed link. It will be the same for everyone who receives the message.

- Use it when the link doesn't change.

**Example:**

- Sending a message about online training. The requirement is that all users visit the same registration page.
  Button Title: Register Now

- Button Value: `https://xyz.org/register`

- All users will see the button “Register Now” and clicking it will take them to `https://xyz.org/register`

**Dynamic URL**

- A Dynamic URL changes for each user. Personalize the link by adding contact-specific data (like name, ID, or custom fields).

- Use it when the link includes unique details per user.

## Creating an HSM Template

1. From the **Templates** page, select **+ Create**.

The **Create HSM Template** page opens with a form on the left and a **Live Preview** panel on the right. The preview updates as you type.

![Create HSM Template page](/img/hsm-templates/hsm-templates_create_form.png)

### Template Details

2. Select a **Language** from the dropdown. This is the primary language for this template. You can add further language variants after the template is approved.

3. Enter an **Element Name** — a unique ID for this template within your WhatsApp Business Account namespace.

   - Use only lowercase letters, numbers, and underscores (e.g., `order_confirmation`).
   - Glific checks availability as you type and shows **Template ID is available** or **Template ID already taken** in real time.

4. Select a **Category** by choosing one of the category cards.

![Category selection cards](/img/hsm-templates/hsm-templates_create_category.png)

- **Utility** — Account updates, order confirmations, shipping notifications, alerts, and transactional messages.
- **Marketing** — Promotional content, offers, announcements, product launches, and sales campaigns.
- **Authentication** — One-time passwords and verification codes for account security.

5. Enter a **Title** — the name for this template as it appears in Glific (e.g., "OTP", "Appointment Reminder", "Activity Preference"). Make it descriptive of the use case.

:::note
Keep **Title** and **Element Name** distinct to avoid mapping issues.
:::

### Message Content

6. Type your message in the **Message Body** field.

![Message body with toolbar](/img/hsm-templates/hsm-templates_create_body.png)

- Use the toolbar to apply **Bold**, _Italic_, or ~~Strikethrough~~ formatting.
- Select **+ Add Variable** to insert a placeholder such as `{{1}}`. An example value field appears below the body for each variable — fill these in to see how the final message looks in the preview.
- Select **AI Assist** to generate message suggestions.
- The character counter shows your progress toward the 1,024-character limit.

7. Optionally enter a **Footer** — a short subtext of up to 60 characters (e.g., "Powered by YourOrg | Reply STOP to unsubscribe").

### Interactive Buttons (optional)

8. In the **Interactive Buttons** card, select a button type if your template needs one.

![Interactive button type selection](/img/hsm-templates/hsm-templates_create_buttons.png)

- **Call to Action** — links to a phone number or URL (static or dynamic).
- **Quick Reply** — lets contacts respond with a single tap (up to 10 buttons).
- **WhatsApp Form** — links to a WhatsApp-native form.

**Call to Action — URL options:**

| URL type    | When to use                                 | Example                        |
| ----------- | ------------------------------------------- | ------------------------------ |
| **Static**  | Same link for everyone                      | `https://xyz.org/register`     |
| **Dynamic** | Personalised link per contact using `{{1}}` | `https://xyz.org/report/{{1}}` |

:::note
Phone numbers without a country code (e.g., Exotel numbers) cannot be used in Call to Action buttons. Add them in the message body instead.

Emojis are not allowed in button text — the template will be rejected.
:::

### Media Attachment (optional)

9. In the **Media Attachment** card, select an attachment type.

![Media attachment options](/img/hsm-templates/hsm-templates_create_media.png)

- **Image** — JPG or PNG, max 5 MB.
- **Document** — PDF, max 16 MB.
- **Video** — MP4, max 16 MB.

When you select a type, paste the public URL of the file from your GCS bucket into the **Attachment URL** field. See [Upload files on GCS](https://glific.github.io/docs/docs/Pre%20Onboarding/GCS%20Setup/Upload%20files%20on%20GCS/#upload-media-files-in-the-gcs-bucket) for how to get a public URL.

:::note
Audio and stickers are not supported in HSM templates.

GIFs are not allowed — if you add a GIF as an attachment, the template will be rejected.
:::

### Organisation & Tags (optional)

10. In the **Organisation & Tags** card, select or create a **Tag** to group this template with others. Tags appear as a filter on the **Templates** list page.

### Submit for Approval

11. Select **Submit for Approval**.

The template is sent to WhatsApp for review. Approval can take anywhere from 2 minutes to 48 hours.

12. Return to the **Templates** page and select **Sync HSM** to refresh the status. Once approved, the template status changes to **Approved**.

## Tips to Get Your Template Approved

## Tips to Get Your Template Approved

1. **Avoid generic messages** — "Hi, here are the activities for today."
1. **Be personal and specific** — "Hi @nameofperson, here are the activities assigned to you. Please complete and submit them by end of day. Thank you."
1. Keeping messages personal and interactive with existing beneficiaries helps approval.
1. GCS attachment links must be **public and accessible** for beneficiaries to open.
1. Use **absolute URLs** in template text. Short URLs cause "Invalid format" rejections.
1. **Emojis are not allowed** in button text.
1. **GIFs are not allowed** as attachments.

:::note
Templates must be in a single language. Do not mix languages within one template (e.g., mixing English and Hindi words in the same body).
:::
