## Read Mode

### When to Use

Use **Read Mode** when you need to:

* Send reminders or scheduled content.
* Retrieve event information.
* Create dynamic menus and option lists.
* Look up user information.
* Fetch content maintained by external teams in Google Sheets.

## Add a Google Sheet in Glific

1. Log in to your Glific account.
2. Navigate to **Flows → Google Sheets** from the left navigation panel.
3. Click **+ Create**.
4. Enter the following details:

   * **Function:** Select **Read** from the drop-down.
   * **Sheet Name:** Enter a name for the integration.
   * **Google Sheet URL:** Open the Google Sheet, copy the URL from the browser, and paste it into the field.
5. Click **Save**.

> **Important**
>
> Open the Google Sheet and click **Share** in the top-right corner.
>
> Add the **service account email** (from your JSON key) and grant the appropriate permission:
>
> * **Viewer** – Read-only access
> * **Editor** – Read and write access
>
> Alternatively, if you do not want to add the service account email, change the sheet permission to **Anyone with the link can view**.

Once saved, the linked sheet can be referenced from any flow.

## Prepare Your Google Sheet

The first column **must** have the header:

```text
Key
```

Glific uses this column to identify the row that should be retrieved.

## Configure Read Mode in a Flow

1. Create a new flow or open an existing flow.
2. Add a **Link Google Sheet** node.
3. Select the linked Google Sheet from the drop-down.
4. Choose **Read**.
5. Select the worksheet (tab) from the drop-down.
6. Enter the value that should be used to search the **Key** column.

For example, if the Google Sheet contains daily activities, you can use:

```text
@calendar.current_date
```

as the search key. Glific will look for the matching value in the **Key** column and retrieve data from that row.

## Available Calendar Variables

The following calendar variables can be used while searching the **Key** column:

```text
@calendar.current_date
@calendar.yesterday
@calendar.tomorrow
@calendar.current_day
@calendar.current_month
@calendar.current_year
```

## Google Sheet Date Format

If your **Key** column contains dates, ensure they are formatted as:

```text
DD/MM/YYYY
```

To ensure compatibility:

* Set the Google Sheet **Locale** to **India**, **or**
* Format the date column as **DD/MM/YYYY**.

Navigate to:

```text
Format → Number → Custom Date and Time
```

Then select or create the format:

```text
DD/MM/YYYY
```

## Access Retrieved Values

Once a matching row is found, you can access any column using:

```text
@results.sheet.column_name
```

Replace `column_name` with the actual column header from your Google Sheet.

**Example**

If your sheet contains:

| Key        | Message  | Video       |
| ---------- | -------- | ----------- |
| 25/06/2026 | Welcome! | https://... |

You can access:

```text
@results.sheet.Message
@results.sheet.Video
```

## Read Mode Examples

Common use cases for **Read Mode** include:

* Sending daily messages based on the current date.
* Displaying event schedules.
* Fetching training content maintained by another team.
* Creating dynamic menus from spreadsheet data.
* Looking up user-specific information.
